问：如果想要传入的参数是对象而不是primitive主数据类型会怎样？
答：在Java中所传递的所有东西都是值，但此值是变量所携带的值，若你对方法传入参数，实际上传入的是远程控制的拷贝。

在Head First Java 第79页 有错误的表述：Getter
数据隐藏：将实际变量标记为private，将Getter和Setter标记为public。
你无需初始化实例变量，因为他们本身是有默认值的，如：数字的预设值（包括char） 为0，boolean的预设值为false,而对象引用则为null，这个null是什么意思？代表没有操作对象的远程控制，它是个引用而不是对象。

如何判断类的继承设计是否正确？
使用IS-A测试来检验，如：三角形是一个多边形=====yes
澡盆是一个浴室======no
想确认你的设计是否正确，使用这样的测试来加以测试，如果不合理，表示你的设计有问题。
